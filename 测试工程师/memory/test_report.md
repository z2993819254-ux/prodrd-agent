## 测试通过模板

```markdown
## QA Test Report (PR #XX)

### 自动化测试
- [x] 单元测试 — ✅ XX passed, 0 failed
- [x] 构建检查 — ✅ build success

### 验收标准验证（对照 Issue #YY）
- [x] 条件 1 — ✅ 通过
- [x] 条件 2 — ✅ 通过
- [x] 条件 3 — ✅ 通过

### 回归检查
- [x] 相关功能 A — 不受影响 ✅
- [x] 相关功能 B — 不受影响 ✅

### 结论
✅ **测试通过，可以合并**
```

## 测试不通过模板

```markdown
## QA Test Report (PR #XX) — ❌ 不通过

### 自动化测试
- [x] 单元测试 — ✅ XX passed
- [ ] 集成测试 — ❌ 2 failed

### 失败项

#### ❌ 失败 1: [描述]
- **复现步骤**: 
  1. 做什么
  2. 做什么
- **期望结果**: xxx
- **实际结果**: xxx
- **日志/截图**: 

#### ❌ 失败 2: [描述]
- **复现步骤**: ...

### 结论
❌ **测试不通过**，需要 Dev 修复以上问题后重新提交。
```

## 规则

- 验收标准里的每一条都要测、都要列出来
- 不通过时必须有复现步骤（开发能直接跟着复现）
- 回归检查至少覆盖改动涉及的相关功能
